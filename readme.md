## 基础准备

### 开通流程
1. 告知员工需要完成的步骤。
2. 提供咨询渠道以解决遇到的问题。

### IDE配置手册
1. 针对不同桌面环境和IDE的配置指南。
2. 解决常见问题的步骤。

### 常见问题收集和回答
1. 收集日常使用中的问题。
2. 提供详细的解答文档。

### 内部宣传
1. 创建宣传账号，发布Copilot相关材料。
2. 整理和分享LLM在研发领域的文章。

### 代理服务器搭建（可选）
1. 搭建日志收集系统。
2. 丰富日志收集内容。

### GH Copilot使用数据展示
1. 搭建GH Copilot Metrics Viewer站点。
2. 收集使用数据，分析用户行为。
3. 提供数据报告，方便后续决策。


## 日常运营方式

### 推广大使
1. 在每个研发团队内部选择一到两个推广大使。
2. 大使作为问题反馈和技术推广的软性负责人。

### 内部分享
1. 不定期的内部使用技巧分享会。
2. 论坛、讨论组等形式的讨论机制。
3. 内部宣传账号的邀稿、投稿。
4. 组织内部积累使用gh copilot的提示词库。

### 外部培训
1. 厂商定期的培训。
2. 厂商的不定期答疑。

### 使用数据分析
1. 参考GitHub Copilot的客观数据。
2. 做企业内部的生产力提升数据分析。
3. 针对不同团队做抽样调查，找到使用的好或不好的原因。

### 定期检查点
1. 跟踪GitHub Copilot的功能更新组织培训。
2. 定期收集用户反馈，了解使用情况。
3. 定期检查和更新使用数据分析。

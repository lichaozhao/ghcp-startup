# 管理员指南

## 邀请用户
1. 收集用户的邮箱地址。
2. 获得管理员的个人访问令牌（PAT）。需要指定PAT有足够的权限。需要admin:org 和 manage_billing:copilot 权限。
3. 使用gh-org-invitation.html工具邀请用户。

## 授权用户
1. 登录 GitHub 组织账户。
2. 进入组织设置页面。
3. 在左侧菜单中选择“成员”。
4. 找到需要授权的用户，点击其旁边的“...”按钮。
5. 选择“管理权限”，然后选择适当的权限级别。

## 删除用户
1. 登录 GitHub 组织账户。
2. 进入组织设置页面。
3. 在左侧菜单中选择“成员”。
4. 找到需要删除的用户，点击其旁边的“...”按钮。
5. 选择“移除成员”。

## 管理个人访问令牌（PAT）
1. 登录 GitHub 账户。
2. 进入个人设置页面。
3. 在左侧菜单中选择“开发者设置”。
4. 选择“个人访问令牌”。
5. 点击“生成新令牌”，选择需要的权限，然后点击“生成令牌”。
6. 复制生成的令牌并妥善保存。

## 访问API
GitHub 提供了丰富的 API 接口，管理员可以通过这些接口管理组织和用户。访问 API 的步骤如下：
1. 生成个人访问令牌（PAT），参考上文。
2. 使用 HTTP 客户端（如 curl 或 Postman）发送 API 请求。
3. 在请求头中添加 `Authorization: token YOUR_PAT`。

## 管理和查看用户使用情况的API
1. 根据gihub copilot metrics viewer搭建服务器，需要配置PAT，组织名称等信息。
2. 使用 GitHub API 获取用户的使用数据。
3. 通过分析数据，了解用户的使用情况和行为模式。
